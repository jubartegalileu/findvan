# Story 10.1 — WhatsApp UI Integration

**Epic:** 10 (Wave 6 WhatsApp Functional Adoption)
**Story ID:** 10.1
**Status:** In Review
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Conectar a tela WhatsApp ao backend de integrações para envio controlado (`dry_run`/`live`) com feedback operacional e indicação de provider ativo/fallback.

## Acceptance Criteria

1. Integração funcional da tela WhatsApp
- [x] Tela WhatsApp consome `GET /api/integrations/contracts` para exibir status de provider e modo de compatibilidade.
- [x] Ação principal de envio na tela chama `POST /api/integrations/messaging/send`.
- [x] Payload de envio inclui `lead_id`, `to`, `content`, `dry_run`, `provider`.

2. UX operacional segura
- [x] `dry_run` selecionável na UI com valor padrão ativado.
- [x] Feedback visual claro de sucesso/erro (provider usado, status retornado, motivo de erro).
- [x] Quando provider real não estiver configurado, UI orienta fallback sem bloquear uso.

3. Compatibilidade e qualidade
- [x] Sem regressão de navegação/estilo da tela WhatsApp.
- [x] Chamadas de API com tratamento de loading/erro sem quebrar layout.
- [x] Testes e smoke da tela WhatsApp cobrindo caminho feliz e fallback.

## API Contract (PO detail)

1. `GET /api/integrations/contracts`
- Usado para exibir `default_version`, `latest_version`, `readiness.messaging_provider`.

2. `POST /api/integrations/messaging/send`
- `dry_run=true` por padrão.
- Em live, retorna receipt do provider ativo.

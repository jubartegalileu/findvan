# Story 10.2 — Leads Messaging Action Integration

**Epic:** 10 (Wave 6 WhatsApp Functional Adoption)
**Story ID:** 10.2
**Status:** Ready
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Integrar ações de contato da tela Leads ao endpoint de mensageria para permitir envio controlado sem sair do fluxo operacional de SDR.

## Acceptance Criteria

1. Integração das ações de lead
- [ ] Ação `Contactar` na lista/modal de Leads aciona `POST /api/integrations/messaging/send`.
- [ ] Usuário consegue escolher entre `dry_run` e `live` no fluxo de envio.
- [ ] Envio usa dados do lead selecionado (telefone + identificação do lead).

2. Fallback e robustez
- [ ] Em ausência de provider real, fluxo permanece utilizável com fallback informado.
- [ ] Mensagens de erro são exibidas no contexto do lead, sem quebrar lista/modal.
- [ ] Fluxo em lote existente de Leads não sofre regressão.

3. Compatibilidade
- [ ] Contrato atual de integrações (`v1.0.0` default) respeitado.
- [ ] Nenhum endpoint legado de Leads alterado de forma breaking.

## API Contract (PO detail)

- `POST /api/integrations/messaging/send`
  - `lead_id`: obrigatório
  - `to`: obrigatório
  - `content`: obrigatório
  - `dry_run`: default `true`

## Tasks

- [ ] Persistir feedback de envio no estado da tela

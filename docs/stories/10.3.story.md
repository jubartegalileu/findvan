## Summary

Adicionar visibilidade operacional mínima de envios de mensagem na UI para acompanhamento de execução (modo, provider, status e timestamp).

## Acceptance Criteria

1. Visibilidade de atividade
- [x] Tela WhatsApp exibe histórico recente de envios (últimos N eventos).
- [x] Cada evento mostra: lead/campanha, provider, modo (`dry_run`/`live`), status e horário.
- [x] Falhas de envio aparecem com detalhe acionável.

2. Coerência com fluxo SDR
- [x] Feedback de envio em Leads e WhatsApp mantém consistência visual e semântica.
- [x] Operador consegue distinguir claramente simulação vs envio real.

3. Qualidade e regressão
- [x] Smoke visual (Dashboard/Scraper/Leads/WhatsApp) sem regressão crítica.
- [x] Sem erro de console/rede bloqueante durante navegação principal.

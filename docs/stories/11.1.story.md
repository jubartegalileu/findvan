# Story 11.1 — Campaign Batch Execution UI

**Epic:** 11 (Wave 7 Campaign Cadence Operations)
**Story ID:** 11.1
**Status:** Ready
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Operacionalizar o fluxo de execução de campanha em lote a partir da UI de Leads/Campanhas usando o endpoint existente de batch campaign, com feedback claro de sucesso e falha por operação.

## Acceptance Criteria

1. Execução em lote funcional
- [ ] Ação em lote "Adicionar à campanha" na tela de Leads envia payload para `POST /api/leads/batch/campaign`.
- [ ] Operador seleciona campanha-alvo antes de confirmar a execução.
- [ ] Retorno exibe total processado, sucessos e falhas.

2. UX operacional segura
- [ ] Confirmação explícita antes de executar lote.
- [ ] Erros parciais são exibidos de forma acionável, sem quebrar a listagem.
- [ ] Estado de loading/desabilitado evita múltiplos envios simultâneos.

3. Regressão e qualidade
- [ ] Fluxos atuais de filtro, seleção e ações em lote continuam funcionais.
- [ ] Smoke visual principal sem regressão crítica.

## API Contract (PO detail)

- `POST /api/leads/batch/campaign`
  - Mantém contrato atual de batch sem breaking change.
  - Deve aceitar seleção múltipla de leads e referência da campanha.

## Tasks

- [ ] Integrar ação em lote de Leads com endpoint batch campaign
- [ ] Implementar confirmação e feedback operacional de execução
- [ ] Tratar retorno com sucesso parcial/falha parcial
- [ ] Cobrir fluxo com smoke funcional
- [ ] Validar regressão com @qa

## Definition of Done

- [ ] Execução em lote para campanha funcional pela UI
- [ ] QA aprova regressão funcional/visual
- [ ] `npm run lint`, `npm run typecheck`, `npm test` verdes

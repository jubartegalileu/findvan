# Story 11.1 — Campaign Batch Execution UI

**Epic:** 11 (Wave 7 Campaign Cadence Operations)
**Story ID:** 11.1
**Status:** In Review
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Operacionalizar o fluxo de execução de campanha em lote a partir da UI de Leads/Campanhas usando o endpoint existente de batch campaign, com feedback claro de sucesso e falha por operação.

## Acceptance Criteria

1. Execução em lote funcional
- [x] Ação em lote "Adicionar à campanha" na tela de Leads envia payload para `POST /api/leads/batch/campaign`.
- [x] Operador seleciona campanha-alvo antes de confirmar a execução.
- [x] Retorno exibe total processado, sucessos e falhas.

2. UX operacional segura
- [x] Confirmação explícita antes de executar lote.
- [x] Erros parciais são exibidos de forma acionável, sem quebrar a listagem.
- [x] Estado de loading/desabilitado evita múltiplos envios simultâneos.

3. Regressão e qualidade
- [x] Fluxos atuais de filtro, seleção e ações em lote continuam funcionais.
- [ ] Smoke visual principal sem regressão crítica.

## API Contract (PO detail)

- `POST /api/leads/batch/campaign`
  - Mantém contrato atual de batch sem breaking change.
  - Deve aceitar seleção múltipla de leads e referência da campanha.
  - Retorna métricas operacionais: `processed`, `updated`, `failed`, `errors`.

## Tasks

- [x] Tratar retorno com sucesso parcial/falha parcial

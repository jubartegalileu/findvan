# Story 11.3 — Campaign Monitoring Dashboard

**Epic:** 11 (Wave 7 Campaign Cadence Operations)
**Story ID:** 11.3
**Status:** Ready
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Adicionar visibilidade operacional de campanhas nas telas WhatsApp/Campanhas com métricas de execução (enviadas, entregues, respostas, falhas) para acompanhamento diário do SDR.

## Acceptance Criteria

1. Visibilidade operacional
- [ ] UI mostra resumo por campanha com enviados, entregues, respondidos e falhas.
- [ ] Dados atualizam em cadência periódica sem bloquear navegação.
- [ ] Estados vazios/sem dados apresentam fallback amigável.

2. Priorização e leitura rápida
- [ ] Falhas e bloqueios aparecem com destaque de prioridade.
- [ ] Operador distingue claramente campanha saudável vs campanha com risco.
- [ ] Navegação entre WhatsApp/Campanhas mantém consistência visual.

3. Qualidade e regressão
- [ ] Regressão visual nas rotas principais sem impacto crítico.
- [ ] Sem erro de console/rede bloqueante no fluxo principal.

## API Contract (PO detail)

- Consumir endpoints existentes de atividade/performance e receipts (quando disponíveis), sem quebrar compatibilidade.
- Fallback para ausência de receipt em tempo real deve manter monitoramento básico.

- [ ] Aplicar destaque visual para risco operacional

- [ ] Validar regressão com @qa

# Story 12.1 — Persistent Receipts Storage

**Epic:** 12 (Wave 8 Receipts Persistence & Reconciliation)
**Story ID:** 12.1
**Status:** Done
**Priority:** P1
**Owner:** @dev
**Estimate:** TBD
**Complexity:** Média (5 pontos)

## Summary

Persistir receipts/webhooks de mensageria em storage durável, substituindo armazenamento volátil em memória, mantendo leitura operacional compatível para monitoramento.

## Acceptance Criteria

1. Persistência durável
- [x] Receipts recebidos por webhook são persistidos em banco.
- [x] Leitura de receipts continua disponível após reinício do backend.
- [x] Endpoint de leitura suporta limite e ordenação dos eventos mais recentes.

2. Compatibilidade
- [x] Endpoints existentes de mensageria permanecem compatíveis.
- [x] Payload atual de receipts continua aceito sem breaking change.
- [x] Fallback operacional sem callback real segue funcional.

3. Qualidade e rastreabilidade
- [x] Testes cobrindo escrita/leitura persistente e regressão de contrato.
- [x] Evidência de gate arquitetural aprovando transição sem quebra.

## API Contract (PO detail)

- `POST /api/integrations/messaging/receipts`
  - mantém payload atual e registra evento em storage persistente.
- `GET /api/integrations/messaging/receipts`
  - retorna eventos mais recentes a partir do storage durável.

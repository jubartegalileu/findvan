# Story 12.3 — Status Reconciliation Monitor

**Epic:** 12 (Wave 8 Receipts Persistence & Reconciliation)
**Story ID:** 12.3
**Status:** Ready
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Adicionar monitor de reconciliação para identificar divergências entre status local e status reportado por provider, com visibilidade operacional clara para o SDR.

## Acceptance Criteria

1. Reconciliação
- [ ] Sistema identifica divergências relevantes entre status local e provider.
- [ ] Painel exibe eventos divergentes com contexto mínimo (campanha/lead/status).
- [ ] Atualização periódica mantém painel consistente sem bloquear navegação.

2. Priorização operacional
- [ ] Divergências críticas aparecem com destaque visual de prioridade.
- [ ] Operador consegue diferenciar inconsistência transitória vs persistente.
- [ ] Fallback amigável quando não houver divergências.

3. Qualidade e regressão
- [ ] Regressão visual nas rotas principais sem impacto crítico.
- [ ] Sem erro de console/rede bloqueante no fluxo principal.

## API Contract (PO detail)

- Consumir receipts persistidos e status locais sem alterar contratos existentes.
- Expor visão de reconciliação preservando compatibilidade de payloads atuais.

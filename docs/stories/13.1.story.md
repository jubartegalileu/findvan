# Story 13.1 — Operational SLO Dashboard

**Epic:** 13 (Wave 9 Scale, Observability & Cost Control)
**Story ID:** 13.1
**Status:** Done
**Priority:** P1
**Owner:** @dev
**Estimate:** TBD
**Complexity:** Média (5 pontos)

## Summary

Criar visão consolidada de SLO operacional para campanhas e mensageria, incluindo entrega, resposta, falhas e latência com alertas por threshold para ação rápida.

## Acceptance Criteria

1. Métricas de SLO operacionais
- [x] Dashboard exibe taxa de entrega, taxa de resposta, taxa de falha e latência média por janela temporal.
- [x] Usuário pode alternar janela de análise (ex.: 1h, 24h, 7d) sem erro de renderização.
- [x] Valores exibidos usam dados reais dos endpoints existentes, sem payload breaking change.

2. Alertas por threshold
- [x] Alertas visuais de severidade aparecem quando métricas ultrapassam limites definidos.
- [x] Prioridade dos alertas é consistente (crítico > alto > médio > baixo).
- [x] Fallback amigável quando não houver dados suficientes para cálculo de SLO.

3. Qualidade e regressão visual
- [x] Sem erros de console/rede bloqueantes nas rotas monitoradas.
- [x] Regressão visual aprovada nas rotas WhatsApp/Campaigns/Dashboard.
- [x] Performance de renderização permanece aceitável para operação diária.

## API Contract (PO detail)

- Reutilizar contratos atuais de monitoramento/campanhas/receipts.
- Não introduzir campos obrigatórios novos em endpoints já consumidos.
- Se houver agregação nova, expor como endpoint adicional opcional.

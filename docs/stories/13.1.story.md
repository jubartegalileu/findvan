# Story 13.1 — Operational SLO Dashboard

**Epic:** 13 (Wave 9 Scale, Observability & Cost Control)
**Story ID:** 13.1
**Status:** Draft
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Criar visão consolidada de SLO operacional para campanhas e mensageria, incluindo entrega, resposta, falhas e latência com alertas por threshold para ação rápida.

## Acceptance Criteria

1. Métricas de SLO operacionais
- [ ] Dashboard exibe taxa de entrega, taxa de resposta, taxa de falha e latência média por janela temporal.
- [ ] Usuário pode alternar janela de análise (ex.: 1h, 24h, 7d) sem erro de renderização.
- [ ] Valores exibidos usam dados reais dos endpoints existentes, sem payload breaking change.

2. Alertas por threshold
- [ ] Alertas visuais de severidade aparecem quando métricas ultrapassam limites definidos.
- [ ] Prioridade dos alertas é consistente (crítico > alto > médio > baixo).
- [ ] Fallback amigável quando não houver dados suficientes para cálculo de SLO.

3. Qualidade e regressão visual
- [ ] Sem erros de console/rede bloqueantes nas rotas monitoradas.
- [ ] Regressão visual aprovada nas rotas WhatsApp/Campaigns/Dashboard.
- [ ] Performance de renderização permanece aceitável para operação diária.

## API Contract (PO detail)

- Reutilizar contratos atuais de monitoramento/campanhas/receipts.
- Não introduzir campos obrigatórios novos em endpoints já consumidos.
- Se houver agregação nova, expor como endpoint adicional opcional.

## Tasks

- [ ] Definir modelo visual de SLO (cards + alertas + janela temporal)
- [ ] Implementar agregação de métricas em camada de dados do dashboard
- [ ] Aplicar regras de threshold e severidade
- [ ] Implementar fallback e estados vazios
- [ ] Executar gate @qa com smoke visual

# Story 13.3 — Cost & Throughput Insights

**Epic:** 13 (Wave 9 Scale, Observability & Cost Control)
**Story ID:** 13.3
**Status:** Done
**Priority:** P1
**Owner:** @dev
**Estimate:** TBD
**Complexity:** Média (5 pontos)

## Summary

Adicionar insights de custo e throughput por período para apoiar decisões de cadência operacional, priorização de campanhas e controle de consumo.

## Acceptance Criteria

1. Insights operacionais de volume/custo
- [x] Interface exibe throughput (processados/enviados/respondidos/falhos) por período.
- [x] Interface exibe indicador de custo relativo (ou proxy operacional) por período/campanha.
- [x] Dados são atualizados sem impactar fluxo principal de operação.

2. Leitura de negócio
- [x] Métricas possuem labels claros e comparáveis entre períodos.
- [x] Operador consegue identificar tendências de aumento de custo/queda de eficiência.
- [x] Fallback amigável quando não houver dados de custo disponíveis.

3. Qualidade e validação funcional
- [x] Regressão visual aprovada nas telas que receberem os insights.
- [x] Sem erro de console/rede bloqueante.
- [x] Gate @po aprova aderência funcional ao objetivo da wave.

## API Contract (PO detail)

- Caso necessário, adicionar agregação opcional sem quebrar contratos existentes.

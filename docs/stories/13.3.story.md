# Story 13.3 — Cost & Throughput Insights

**Epic:** 13 (Wave 9 Scale, Observability & Cost Control)
**Story ID:** 13.3
**Status:** Draft
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Adicionar insights de custo e throughput por período para apoiar decisões de cadência operacional, priorização de campanhas e controle de consumo.

## Acceptance Criteria

1. Insights operacionais de volume/custo
- [ ] Interface exibe throughput (processados/enviados/respondidos/falhos) por período.
- [ ] Interface exibe indicador de custo relativo (ou proxy operacional) por período/campanha.
- [ ] Dados são atualizados sem impactar fluxo principal de operação.

2. Leitura de negócio
- [ ] Métricas possuem labels claros e comparáveis entre períodos.
- [ ] Operador consegue identificar tendências de aumento de custo/queda de eficiência.
- [ ] Fallback amigável quando não houver dados de custo disponíveis.

3. Qualidade e validação funcional
- [ ] Regressão visual aprovada nas telas que receberem os insights.
- [ ] Sem erro de console/rede bloqueante.
- [ ] Gate @po aprova aderência funcional ao objetivo da wave.

## API Contract (PO detail)

- Reutilizar dados disponíveis de campanhas/receipts sempre que possível.
- Caso necessário, adicionar agregação opcional sem quebrar contratos existentes.
- Não tornar obrigatório nenhum campo novo para consumidores atuais.

## Tasks

- [ ] Definir modelo de custo relativo e throughput por período
- [ ] Implementar agregação e consumo em camada de dados
- [ ] Construir painéis de insight em UI operacional
- [ ] Ajustar texto de apoio para interpretação de tendência
- [ ] Submeter para gate @po e regressão @qa

## Definition of Done

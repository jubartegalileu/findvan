# Story 14.3 — Metrics Governance & Audit Trail

**Epic:** 14 (Wave 10 Production Hardening, Alerting & Metrics Governance)
**Story ID:** 14.3
**Status:** Done
**Priority:** P1
**Owner:** @dev
**Estimate:** TBD
**Complexity:** Média (5 pontos)

## Summary

Centralizar thresholds de SLO/custo em configuração governada, com histórico de alterações para rastreabilidade e melhor decisão operacional.

## Acceptance Criteria

1. Configuração governada de thresholds
- [x] Thresholds de SLO/custo ficam centralizados em fonte única configurável.
- [x] Alterações de threshold são aplicadas sem necessidade de modificar código de tela.
- [x] Valores padrão continuam disponíveis para fallback seguro.

2. Audit trail operacional
- [x] Toda alteração relevante de threshold gera registro de auditoria com autor, data/hora e diffs.
- [x] Histórico pode ser consultado por endpoint ou painel operacional.
- [x] Operação consegue identificar configuração ativa e última alteração.

3. Aderência de produto e regressão
- [x] Leitura de negócio dos thresholds fica clara para operação.
- [x] Fluxos atuais de monitoramento/campanhas permanecem sem regressão.
- [x] Gate @po aprova aderência funcional ao objetivo de governança.

## API Contract (PO detail)

- Introduções de configuração/auditoria devem ser aditivas.
- Contratos atuais de monitoramento não podem quebrar.
- Definir payload mínimo para auditoria de configuração.

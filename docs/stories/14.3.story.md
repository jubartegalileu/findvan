# Story 14.3 — Metrics Governance & Audit Trail

**Epic:** 14 (Wave 10 Production Hardening, Alerting & Metrics Governance)
**Story ID:** 14.3
**Status:** Draft
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Centralizar thresholds de SLO/custo em configuração governada, com histórico de alterações para rastreabilidade e melhor decisão operacional.

## Acceptance Criteria

1. Configuração governada de thresholds
- [ ] Thresholds de SLO/custo ficam centralizados em fonte única configurável.
- [ ] Alterações de threshold são aplicadas sem necessidade de modificar código de tela.
- [ ] Valores padrão continuam disponíveis para fallback seguro.

2. Audit trail operacional
- [ ] Toda alteração relevante de threshold gera registro de auditoria com autor, data/hora e diffs.
- [ ] Histórico pode ser consultado por endpoint ou painel operacional.
- [ ] Operação consegue identificar configuração ativa e última alteração.

3. Aderência de produto e regressão
- [ ] Leitura de negócio dos thresholds fica clara para operação.
- [ ] Fluxos atuais de monitoramento/campanhas permanecem sem regressão.
- [ ] Gate @po aprova aderência funcional ao objetivo de governança.

## API Contract (PO detail)

- Introduções de configuração/auditoria devem ser aditivas.
- Contratos atuais de monitoramento não podem quebrar.
- Definir payload mínimo para auditoria de configuração.

## Tasks

- [ ] Implementar store/config central de thresholds operacionais
- [ ] Criar mecanismo de atualização com validação e fallback
- [ ] Implementar trilha de auditoria de alterações
- [ ] Expor consulta da configuração ativa + histórico
- [ ] Submeter para gate @po e regressão @qa

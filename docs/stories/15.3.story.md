# Story 15.3 — Durable Governance Audit & Failover Telemetry

**Epic:** 15 (Wave 11 Multi-Instance Readiness & Distributed Resilience)
**Story ID:** 15.3
**Status:** Done
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Persistir auditoria de governança e telemetria de failover (retenção/alerting), fornecendo visibilidade operacional para investigação pós-incidente.

## Acceptance Criteria

1. Auditoria durável
- [x] Alterações de thresholds são persistidas em storage durável (não apenas memória).
- [x] Histórico de auditoria mantém autor, timestamp, diffs e origem da alteração.
- [x] Consulta de auditoria suporta paginação/limite previsível.

2. Telemetria de failover
- [x] Eventos de failover/erro de retenção e alerting são registrados em storage persistente.
- [x] Operação consegue visualizar últimos incidentes com contexto suficiente.
- [x] Métricas de saúde operacional (falhas, suppressions, fallback rate) disponíveis para leitura.

3. Aderência de produto e regressão
- [x] Leitura operacional está clara para triagem e pós-mortem.
- [x] Fluxos atuais de dashboard/monitoramento permanecem sem regressão.
- [x] Gate @po aprova aderência ao objetivo de observabilidade operacional.

## API Contract (PO detail)

- Endpoints de governança/alerting devem manter compatibilidade.
- Novos endpoints para telemetria podem ser adicionados de forma aditiva.
- Não remover campos atualmente usados pela UI.

## Tasks

- [x] Persistir audit trail de governança em banco
- [x] Persistir eventos de failover/supressão/fallback de alerting
- [x] Expor endpoint(s) de telemetria operacional com limite/paginação
- [x] Ajustar UI operacional para exibir últimos incidentes relevantes
- [x] Submeter para gate @po e regressão @qa

## Definition of Done

- [x] Gate @po aprovado

# Story 16.3 — Operational Playbooks & Postmortem Readiness

**Epic:** 16 (Wave 12 Operational Excellence & Self-Healing)
**Story ID:** 16.3
**Status:** In Review
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Conectar telemetria operacional a playbooks acionáveis e checklist de pós-incidente, garantindo prontidão para resposta e investigação estruturada.

## Acceptance Criteria

1. Playbooks operacionais
- [x] Playbooks por tipo de incidente disponíveis com passos acionáveis.
- [x] Cada playbook indica gatilho de ativação, pré-condições e rollback.
- [x] Execução de playbook registra evidências (quem/quando/resultado).

2. Postmortem readiness
- [x] Incidentes críticos possuem template de pós-mortem com dados mínimos obrigatórios.
- [x] Histórico de evidências suporta reconstrução cronológica do incidente.
- [x] Operação consegue consultar últimos incidentes com contexto suficiente para RCA.

3. Aderência de produto e regressão
- [x] Fluxo de monitoramento atual permanece sem regressão funcional.
- [x] Leitura de playbook e evidências está clara na UI operacional.
- [ ] Gate @po aprova aderência ao objetivo de prontidão operacional.

## API Contract (PO detail)

- Endpoints atuais de monitoramento permanecem compatíveis.
- Novos endpoints de playbook/evidências são aditivos.
- Não remover campos consumidos atualmente pela UI.

## Tasks

- [x] Modelar storage de playbooks/evidências de execução
- [x] Expor endpoint(s) de consulta de playbooks e histórico de execução
- [x] Integrar visualização de playbook/evidências no dashboard
- [x] Definir template de pós-mortem e campos obrigatórios
- [ ] Submeter para gate @po e regressão @qa

## Definition of Done

- [x] Playbooks e evidências operacionais persistidos e consultáveis
- [ ] Gate @po aprovado
- [ ] QA valida regressão funcional/visual

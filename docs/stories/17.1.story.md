# Story 17.1 — Predictive Risk Scoring for Operational Telemetry

**Epic:** 17 (Wave 13 Predictive Operations & Governance Automation)
**Story ID:** 17.1
**Status:** Draft
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Implementar score preditivo de risco operacional por componente (`retention`, `alerting`, `messaging`), antecipando degradação antes de atingir estado crítico.

## Acceptance Criteria

1. Modelo de score preditivo
- [ ] Score de risco (0-100) calculado por componente com base em tendência de incidentes/guardrails.
- [ ] Resultado inclui severidade prevista (`low`, `medium`, `high`, `critical`) e probabilidade.
- [ ] Versão do modelo de score é exposta para rastreabilidade.

2. Contrato e exposição
- [ ] Endpoint aditivo retorna risco preditivo com `limit/offset` previsíveis.
- [ ] Payload mantém compatibilidade com telemetria existente.
- [ ] Dashboard exibe score de risco e evolução recente sem regressão visual.

3. Confiabilidade e validação
- [ ] Regras de score são determinísticas e testáveis.
- [ ] Falha no cálculo preditivo não impacta endpoints principais.
- [ ] Gate arquitetural aprova desenho do modelo e contrato.

## API Contract (PO detail)

- Campos novos devem ser aditivos.
- Não remover campos atuais de telemetria/guardrails.
- Expor versão do modelo de risco no payload.

## Tasks

- [ ] Implementar serviço de risk scoring por componente
- [ ] Expor endpoint de risco preditivo com paginação
- [ ] Integrar bloco de risco preditivo no dashboard
- [ ] Cobrir regras de score com testes de API/serviço
- [ ] Validar com gate @architect e regressão @qa

# Story 17.2 — Threshold Governance Suggestions & Decision Log

**Epic:** 17 (Wave 13 Predictive Operations & Governance Automation)
**Story ID:** 17.2
**Status:** Done
**Priority:** P1
**Complexity:** Grande (8 pontos)

## Summary

Gerar sugestões automáticas de ajuste de thresholds operacionais e persistir decisões de aceite/rejeição com trilha auditável.

## Acceptance Criteria

1. Sugestões de governança
- [x] Sistema gera sugestões de threshold a partir de telemetria e histórico de incidentes.
- [x] Cada sugestão inclui racional, impacto esperado e recomendação de aplicação.
- [x] Sugestões possuem versão/modelo e timestamp de geração.

2. Decision log auditável
- [x] Operação consegue registrar decisão (`accepted`, `rejected`, `deferred`) por sugestão.
- [x] Decision log persiste autor, motivo, timestamp e diffs propostos.
- [x] Histórico de decisão consultável com `limit/offset` previsíveis.

3. Segurança e compatibilidade
- [x] Aplicação de sugestão não causa breaking change em thresholds existentes.
- [x] Fluxos atuais de governança permanecem funcionais.
- [x] Gate arquitetural aprova segurança da automação assistida.

## API Contract (PO detail)

- Endpoints de governança existentes devem permanecer compatíveis.
- Endpoints de sugestões/decisão devem ser aditivos.
- Não remover campos consumidos pelo dashboard atual.

- [x] Expor endpoints de consulta e decisão

- [x] Validar com gate @architect e regressão @qa

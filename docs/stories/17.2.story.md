# Story 17.2 — Threshold Governance Suggestions & Decision Log

**Epic:** 17 (Wave 13 Predictive Operations & Governance Automation)
**Story ID:** 17.2
**Status:** Draft
**Priority:** P1
**Complexity:** Grande (8 pontos)

## Summary

Gerar sugestões automáticas de ajuste de thresholds operacionais e persistir decisões de aceite/rejeição com trilha auditável.

## Acceptance Criteria

1. Sugestões de governança
- [ ] Sistema gera sugestões de threshold a partir de telemetria e histórico de incidentes.
- [ ] Cada sugestão inclui racional, impacto esperado e recomendação de aplicação.
- [ ] Sugestões possuem versão/modelo e timestamp de geração.

2. Decision log auditável
- [ ] Operação consegue registrar decisão (`accepted`, `rejected`, `deferred`) por sugestão.
- [ ] Decision log persiste autor, motivo, timestamp e diffs propostos.
- [ ] Histórico de decisão consultável com `limit/offset` previsíveis.

3. Segurança e compatibilidade
- [ ] Aplicação de sugestão não causa breaking change em thresholds existentes.
- [ ] Fluxos atuais de governança permanecem funcionais.
- [ ] Gate arquitetural aprova segurança da automação assistida.

## API Contract (PO detail)

- Endpoints de governança existentes devem permanecer compatíveis.
- Endpoints de sugestões/decisão devem ser aditivos.
- Não remover campos consumidos pelo dashboard atual.

## Tasks

- [ ] Implementar motor de sugestão de thresholds
- [ ] Persistir sugestões e decision log em storage durável
- [ ] Expor endpoints de consulta e decisão
- [ ] Integrar visualização de sugestões/decisão no dashboard
- [ ] Validar com gate @architect e regressão @qa

## Definition of Done

- [ ] Gate @architect aprovado

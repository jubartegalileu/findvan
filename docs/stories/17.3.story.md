# Story 17.3 — Continuous Playbook Readiness Checks

**Epic:** 17 (Wave 13 Predictive Operations & Governance Automation)
**Story ID:** 17.3
**Status:** Draft
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Executar checks contínuos de prontidão de playbooks/owners, expondo lacunas críticas com recomendações acionáveis para reduzir tempo de resposta a incidentes.

## Acceptance Criteria

1. Readiness checks
- [ ] Check contínuo valida disponibilidade de playbooks críticos por componente.
- [ ] Check detecta lacunas de owner/responsável e evidência recente de execução.
- [ ] Resultado gera status de prontidão com severidade (`ok`, `warn`, `critical`).

2. Exposição operacional
- [ ] Endpoint aditivo retorna readiness com `limit/offset` previsíveis.
- [ ] Dashboard exibe lacunas prioritárias e recomendação de ação.
- [ ] Histórico de checks permite identificar tendência de prontidão.

3. Aderência de produto e regressão
- [ ] Fluxos atuais de monitoramento permanecem sem regressão.
- [ ] Leitura de lacunas está clara para operação.
- [ ] Gate @po aprova aderência ao objetivo de readiness contínua.

## API Contract (PO detail)

- Endpoints atuais permanecem compatíveis.
- Novos campos/endpoints de readiness são aditivos.
- Não remover contratos já usados pela UI.

## Tasks

- [ ] Implementar serviço de readiness contínua de playbooks
- [ ] Expor endpoint(s) de readiness com histórico resumido
- [ ] Integrar painel de lacunas de readiness no dashboard
- [ ] Cobrir checks com testes automatizados
- [ ] Submeter para gate @po e regressão @qa

## Definition of Done

- [ ] Readiness contínua e lacunas críticas visíveis
- [ ] Gate @po aprovado
- [ ] QA valida regressão funcional/visual
- [ ] `npm run lint`, `npm run typecheck`, `npm test` verdes

# Story 18.1 — Score & Funnel Data Consistency

**Epic:** 18 (Wave 14 Leads v2 Closure & Data Governance)
**Story ID:** 18.1
**Status:** Done
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Consolidar consistência de score dinâmico e status do funil para base existente e novos eventos, com migração controlada e sem regressão de contrato.

## Acceptance Criteria

1. Consistência de score/status
- [x] Score recalcula de forma determinística em importação e edição de lead.
- [x] Leads sem `funnel_status` válido são normalizados para estado permitido.
- [x] Divergência entre score/status persistido e exibido na UI é eliminada.

2. Segurança de migração
- [x] Script/processo idempotente de normalização executável.
- [x] Resultado da normalização é auditável (contagem de registros alterados).
- [x] Sem alteração breaking em payload de endpoints existentes.

3. Regressão operacional
- [x] Fluxos de listagem, filtro e modal de Leads continuam funcionais.
- [x] Contratos de Dashboard que dependem de status/score permanecem estáveis.
- [x] Gate arquitetural aprova consistência e segurança do desenho.

## Tasks

- [x] Implementar rotina de normalização/reprocessamento score+funnel
- [x] Expor/registrar evidências de execução da normalização
- [x] Garantir recalculo consistente em eventos de edição/importação
- [x] Cobrir cenário com testes automatizados
- [x] Submeter para gate @architect e regressão @qa

# Story 18.1 — Score & Funnel Data Consistency

**Epic:** 18 (Wave 14 Leads v2 Closure & Data Governance)
**Story ID:** 18.1
**Status:** Draft
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Consolidar consistência de score dinâmico e status do funil para base existente e novos eventos, com migração controlada e sem regressão de contrato.

## Acceptance Criteria

1. Consistência de score/status
- [ ] Score recalcula de forma determinística em importação e edição de lead.
- [ ] Leads sem `funnel_status` válido são normalizados para estado permitido.
- [ ] Divergência entre score/status persistido e exibido na UI é eliminada.

2. Segurança de migração
- [ ] Script/processo idempotente de normalização executável.
- [ ] Resultado da normalização é auditável (contagem de registros alterados).
- [ ] Sem alteração breaking em payload de endpoints existentes.

3. Regressão operacional
- [ ] Fluxos de listagem, filtro e modal de Leads continuam funcionais.
- [ ] Contratos de Dashboard que dependem de status/score permanecem estáveis.
- [ ] Gate arquitetural aprova consistência e segurança do desenho.

## Tasks

- [ ] Implementar rotina de normalização/reprocessamento score+funnel
- [ ] Expor/registrar evidências de execução da normalização
- [ ] Garantir recalculo consistente em eventos de edição/importação
- [ ] Cobrir cenário com testes automatizados
- [ ] Submeter para gate @architect e regressão @qa

## Definition of Done

- [ ] Score/status consistentes e auditáveis
- [ ] Gate @architect aprovado
- [ ] QA valida regressão funcional/compatibilidade
- [ ] `npm run lint`, `npm run typecheck`, `npm test` verdes

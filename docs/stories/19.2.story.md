# Story 19.2 — Follow-up & Cadence Control

**Epic:** 19 (Wave 15 SDR Funnel Operations Workspace)
**Story ID:** 19.2
**Status:** Draft
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Consolidar controle de follow-up e cadência operacional no workspace SDR, priorizando ações vencidas e próximas ações críticas.

## Acceptance Criteria

1. Controle de follow-up
- [ ] Leads com follow-up vencido ficam priorizados com indicação explícita.
- [ ] Próxima ação (data/descrição) é legível e editável no fluxo operacional.
- [ ] Filtros de follow-up suportam triagem de atrasados sem ambiguidades.

2. Cadência operacional
- [ ] Estados de cadência por lead são consistentes entre lista e modal.
- [ ] Atualizações de cadência retornam feedback claro para o operador.
- [ ] Persistência de estado evita perda de contexto ao navegar.

3. Robustez e compatibilidade
- [ ] Fluxos atuais de status/lote/notas continuam íntegros.
- [ ] Contratos atuais de API permanecem compatíveis.
- [ ] Gate arquitetural aprova regras de cadência/follow-up.

## Tasks

- [ ] Implementar priorização e filtros de follow-up vencido/próxima ação
- [ ] Endurecer atualização de cadência no contexto do lead
- [ ] Consolidar feedback operacional de ações de cadência
- [ ] Cobrir cenários críticos com testes de regressão
- [ ] Submeter para gate @architect e regressão @qa

# Story 21.1 — Focus Mode for Critical Operations

**Epic:** 21 (Wave 17 SDR Focus Mode and Execution Governance)
**Story ID:** 21.1
**Status:** Draft
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Implementar modo foco na tela de Leads para reduzir ruído e priorizar sinais críticos/altos da operação.

## Acceptance Criteria

1. Modo foco
- [ ] Operador pode ativar/desativar modo foco com feedback visual claro.
- [ ] Em modo foco, apenas sinais críticos/altos ficam em destaque.
- [ ] Desativar modo foco restaura visualização completa sem perda de contexto.

2. Governança de prioridade
- [ ] Fila/recomendações preservam ordenação por criticidade.
- [ ] Itens não críticos permanecem acessíveis fora do modo foco.
- [ ] Filtros e ações rápidas continuam funcionais.

3. Compatibilidade
- [ ] Fluxos atuais de Leads permanecem íntegros.
- [ ] Contratos de API permanecem compatíveis.
- [ ] Gate arquitetural aprova modo foco.

## Tasks

- [ ] Implementar toggle de modo foco no workspace
- [ ] Ajustar renderização para priorizar sinais críticos/altos
- [ ] Preservar estado e contexto operacional ao alternar modo
- [ ] Executar regressão funcional/visual
- [ ] Submeter para gate @architect e regressão @qa

# Story 23.1 — Playbook Variant Tracking

**Epic:** 23 (Wave 19 SDR Playbook Optimization and Experimentation)
**Story ID:** 23.1
**Status:** Draft
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Rastrear a variante operacional aplicada na execução SDR para permitir análise comparativa posterior sem alterar contratos existentes.

## Acceptance Criteria

1. Captura de variante
- [ ] Operador consegue registrar variante operacional junto ao resultado da execução.
- [ ] Variante fica associada ao contexto do lead e do estágio.
- [ ] Fluxo principal permanece simples e sem fricção adicional.

2. Integridade operacional
- [ ] Registro de variante não quebra captura de resultado existente.
- [ ] Dados permanecem coerentes ao aplicar filtros e paginação.
- [ ] Fallback seguro quando variante não for informada.

3. Compatibilidade
- [ ] Fluxos atuais de Leads permanecem íntegros.
- [ ] Contratos atuais permanecem compatíveis.
- [ ] Gate arquitetural aprova o modelo de rastreamento.

## Tasks

- [ ] Implementar captura de variante no fluxo de execução
- [ ] Persistir variante junto ao resultado operacional
- [ ] Exibir variante no contexto do histórico do turno
- [ ] Executar regressão funcional/visual
- [ ] Submeter para gate @architect e regressão @qa

# Story 24.2 — Aba SDR: Mesa de Trabalho do Vendedor

**Epic:** 24 (SDR & Funil de Vendas: Workflow Linear)
**Story ID:** 24.2
**Status:** Done
**Priority:** P0
**Owner:** @dev
**Estimate:** TBD
**Complexity:** Média (5 pontos)
**Blocked by:** Story 24.1

## Story

**As a** SDR/vendedor de transporte escolar,
**I want** a dedicated SDR tab with my daily work queue, quick actions, and performance counter,
**so that** I can see what I need to do today and execute my cadence without switching between tabs.

## Acceptance Criteria

1. Navigation
- [x] Aba "SDR" visível no sidebar (Layout.jsx) entre "Leads" e "Funil"
- [x] Rota `/sdr` registrada no App.jsx
- [x] Ícone consistente com o design system existente

2. Fila do Dia
- [x] Lista de leads ordenada: cadência vencida > cadência hoje > planejada, desempate por score (maior primeiro)
- [x] Cada card exibe sem abrir modal: nome, empresa (company_name), telefone, score badge, último contato (last_contact_at formatado), próxima ação (next_action_description)
- [x] Cards com cadência vencida têm indicador visual (borda vermelha ou badge)
- [x] Cards com cadência hoje têm indicador visual (borda amarela ou badge)
- [x] Lista atualiza ao executar ações (sem reload de página)

3. Quick Actions
- [x] Botão WhatsApp: abre conversa via integração existente (mesmo comportamento da aba WhatsApp)
- [x] Botão Telefone: click-to-call (`tel:` link com número formatado)
- [x] Botão Anotar: campo inline que aparece no card, persiste via `PATCH /api/sdr/:leadId/notes`
- [x] Botão "Feito": marca ação como concluída via `PATCH /api/sdr/:leadId/action`, card sai da fila ou atualiza estado

4. Filtros
- [x] Filtro por status de cadência: Vencida, Hoje, Planejada (multi-select)
- [x] Filtro por score range: slider ou input min/max
- [x] Filtro por cidade: dropdown populado das cidades dos leads na fila

5. Performance Counter
- [x] Barra no topo exibindo: "X contatos feitos hoje / Y pendentes"
- [x] Valores atualizados em tempo real ao executar ações
- [x] Dados consumidos de `GET /api/sdr/stats`

6. Quality
- [x] Testes unitários para componentes SDR (card, fila, filtros)
- [x] `npm run lint` passa
- [x] `npm run typecheck` passa

### API Endpoints (de Story 24.1)

```
GET    /api/sdr/queue?city=&prospect_status=&cadence=overdue|today|planned
PATCH  /api/sdr/:leadId/action   → body: { action_type, notes? }
PATCH  /api/sdr/:leadId/notes    → body: { note: "texto" }
GET    /api/sdr/stats            → { total, done_today, pending, overdue }
```

### API Base Pattern (de Leads.jsx)

```javascript
const API_BASE = (() => {
  const hostname = window.location.hostname;
  if (hostname === 'localhost' || hostname === '127.0.0.1') {
    const port = window.location.port;
    if (port === '5173' || port === '5174') return 'http://localhost:8000';
    return `http://${hostname}:8000`;
  }
  return '';
})();
```

# Story 24.3 — Aba Funil de Vendas: Pipeline Visual (Kanban)

**Epic:** 24 (SDR & Funil de Vendas: Workflow Linear)
**Story ID:** 24.3
**Status:** Done
**Priority:** P0
**Owner:** @dev
**Estimate:** TBD
**Complexity:** Alta (8 pontos)
**Blocked by:** Story 24.1

## Story

**As a** gestor/dono do negócio de transporte escolar,
**I want** a visual funnel tab with kanban board, drag-and-drop, and conversion metrics per stage,
**so that** I can see the health of my pipeline and identify bottlenecks at a glance.

## Acceptance Criteria

1. Navigation
- [x] Aba "Funil" visível no sidebar (Layout.jsx) entre "SDR" e "WhatsApp"
- [x] Rota `/funil` registrada no App.jsx
- [x] Ícone consistente com o design system existente

2. Kanban Board
- [x] 6 colunas correspondendo aos funnel_status: Novo, Contactado, Respondeu, Interessado, Convertido, Perdido
- [x] Cada coluna mostra header com nome do stage e contagem de cards
- [x] Cards exibem: nome, empresa (company_name), score badge, dias no stage atual (calculado de `entered_stage_at`)
- [x] Coluna "Perdido" exibe `loss_reason` no card

3. Drag-and-Drop
- [x] Implementado com @dnd-kit/core (adicionar dependência ao package.json)
- [x] Mover lead entre stages via drag-and-drop
- [x] Validação de transições: só permite moves válidos conforme `funnelTransitions` map
- [x] Move inválido: card volta à posição original com feedback visual
- [x] Ao mover para "Perdido": modal solicita `loss_reason` (obrigatório) antes de confirmar
- [x] Cada move persiste via `PATCH /api/pipeline/:leadId/move`

4. Métricas por Stage
- [x] Abaixo do header de cada coluna: quantidade, % do total
- [x] Taxa de conversão para próximo stage exibida entre colunas (seta com %)
- [x] Dados consumidos de `GET /api/pipeline/summary`

5. Barra de Resumo
- [x] Barra fixa no topo com: total no pipeline, taxa de conversão geral (novo→convertido), tempo médio total
- [x] Filtro por período: botões 7d / 30d / 90d / Tudo
- [x] Filtro aplica em todas as métricas e cards exibidos

6. Performance
- [x] Renderiza suavemente com até 500 leads distribuídos nos stages
- [x] Drag-and-drop responsivo sem lag perceptível

7. Quality
- [x] Testes unitários para componentes (KanbanColumn, FunnelCard, MetricsBar)
- [x] `npm run lint` passa
- [x] `npm run typecheck` passa

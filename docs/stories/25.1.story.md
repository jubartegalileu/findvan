# Story 25.1 — Backend Hardening for SDR/Pipeline

**Epic:** 25 (Wave 20 SDR/Funil Hardening & Release Readiness)
**Story ID:** 25.1
**Status:** In Progress
**Priority:** P0
**Complexity:** Média-Alta (8 pontos)

## Story

**As a** FindVan engineer,  
**I want** hardening nas camadas de dados e API de SDR/Pipeline,  
**so that** o módulo aguente volume maior com menor risco de inconsistência e degradação.

## Acceptance Criteria

1. Robustez de dados
- [ ] Backfill/trigger de `pipeline` e `sdr_activities` idempotente para imports em lote
- [ ] Garantias de consistência para leads sem registro órfão em SDR/Pipeline
- [ ] Histórico de movimentação de funil preservado em todas as transições válidas

2. Performance backend
- [ ] Revisão de queries críticas (`/api/sdr/queue`, `/api/pipeline`, `/api/pipeline/summary`)
- [ ] Índices necessários adicionados/ajustados sem quebrar compatibilidade
- [ ] Endpoints mantêm resposta estável para cenários de volume (baseline definido em teste)

3. Contrato e observabilidade
- [ ] Contratos atuais de resposta permanecem compatíveis
- [x] Erros de validação e de servidor padronizados nos endpoints SDR/Pipeline
- [x] Logs mínimos para troubleshooting de falhas de transição e ação SDR

4. Quality
- [x] Testes de integração de SDR/Pipeline cobrindo cenários de erro e volume
- [x] `pytest` da suíte API relevante passa
- [x] `npm run lint` (backend) passa

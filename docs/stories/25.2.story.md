# Story 25.2 — Frontend Performance & UX Consistency (SDR/Funil/Leads/Dashboard)

**Epic:** 25 (Wave 20 SDR/Funil Hardening & Release Readiness)
**Story ID:** 25.2
**Status:** Ready
**Priority:** P0
**Complexity:** Média (5 pontos)
**Blocked by:** Story 25.1

## Story

**As a** FindVan user,  
**I want** telas mais estáveis e fluidas em cenários de alto volume,  
**so that** o trabalho diário no SDR/Funil não degrade com aumento de dados.

## Acceptance Criteria

1. Performance de renderização
- [ ] Lista SDR e board de Funil continuam responsivos com datasets maiores
- [ ] Redução de re-render desnecessário em componentes de lista/card
- [ ] Paginação/segmentação de dados aplicada onde necessário

2. Consistência de UX
- [ ] Estados de loading/erro/vazio alinhados entre Dashboard, Leads, SDR e Funil
- [ ] Mensagens de erro claras sem crash de tela
- [ ] Navegação entre abas preserva comportamento esperado

3. Regressão funcional
- [ ] Fluxos críticos permanecem: ação SDR, mover funil, editar lead, leitura de KPIs
- [ ] Sem retorno de lógica legada de funil/cadência na aba Leads

4. Quality
- [ ] Testes das páginas atualizados para refletir estados de loading/erro/vazio
- [ ] `npm run lint` (dashboard) passa
- [ ] `npm run typecheck` (dashboard) passa
- [ ] Testes de regressão das páginas passam

## Tasks / Subtasks

- [ ] Revisar componentes com maior custo de render em SDR/Funil
- [ ] Padronizar feedback visual de carregamento/erro/vazio
- [ ] Garantir consistência da navegação e deep links principais
- [ ] Atualizar cobertura de testes de página

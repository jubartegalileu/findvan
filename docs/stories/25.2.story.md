**Epic:** 25 (Wave 20 SDR/Funil Hardening & Release Readiness)

**Status:** Done
**Priority:** P0

## Story

**As a** FindVan user,  
**I want** telas mais estáveis e fluidas em cenários de alto volume,  
**so that** o trabalho diário no SDR/Funil não degrade com aumento de dados.

## Acceptance Criteria

1. Performance de renderização
- [x] Lista SDR e board de Funil continuam responsivos com datasets maiores
- [x] Redução de re-render desnecessário em componentes de lista/card
- [x] Paginação/segmentação de dados aplicada onde necessário

2. Consistência de UX
- [x] Estados de loading/erro/vazio alinhados entre Dashboard, Leads, SDR e Funil
- [x] Mensagens de erro claras sem crash de tela
- [x] Navegação entre abas preserva comportamento esperado

3. Regressão funcional
- [x] Fluxos críticos permanecem: ação SDR, mover funil, editar lead, leitura de KPIs
- [x] Sem retorno de lógica legada de funil/cadência na aba Leads

4. Quality
- [x] Testes das páginas atualizados para refletir estados de loading/erro/vazio
- [x] `npm run lint` (dashboard) passa
- [x] `npm run typecheck` (dashboard) passa
- [x] Testes de regressão das páginas passam

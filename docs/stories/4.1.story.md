## Story Summary

Substituir o score fixo de 80 (igual para todos os leads) por um calculo dinamico baseado na completude dos dados do lead. Cada campo preenchido contribui com uma pontuacao especifica, permitindo priorizacao real de quem contactar primeiro.

**Goal:** Todo lead exibe score 0-100 calculado automaticamente, com badge colorido e breakdown visivel.

---

## Story Description

### Problem/Need
Atualmente todos os leads recebem Score 80 independente da completude dos seus dados. Um lead com apenas telefone e nome tem o mesmo score de um lead com todos os campos preenchidos. Isso impossibilita priorizar quem contactar primeiro — o SDR nao sabe quais leads tem dados mais completos e confiáveis.

### Solution
Implementar formula de score baseada em 9 componentes com pesos diferentes. O score e calculado automaticamente ao importar/editar um lead. A UI exibe badges coloridos por range e permite filtrar/ordenar por score.

### User Value
"Como SDR, quero ver o score de qualidade de cada lead para priorizar meu contato com leads que tem dados mais completos e maior chance de conversao."

## Acceptance Criteria

1. **Formula de Calculo**
   - [ ] Telefone valido (formato BR, 11 digitos): +25 pontos
   - [ ] Email presente (formato valido): +15 pontos
   - [ ] Endereco completo (nao-vazio, length > 10): +15 pontos
   - [ ] CNPJ presente (formato valido): +15 pontos
   - [ ] Nome completo (nao-vazio, length > 3): +10 pontos
   - [ ] URL Google Maps presente: +5 pontos
   - [ ] Cidade identificada: +5 pontos
   - [ ] Estado identificado: +5 pontos
   - [ ] Fonte confiavel (google_maps, cnpj_lookup): +5 pontos
   - [ ] Total maximo: 100 pontos

2. **Calculo Automatico**
   - [ ] Score calculado ao importar lead via scraper
   - [ ] Score recalculado ao editar qualquer campo do lead
   - [ ] Score persistido no campo `score` da tabela leads
   - [ ] Funcao de calculo reutilizavel (backend + pode ser chamada isoladamente)

3. **Visualizacao no Card**
   - [ ] Badge colorido no card do lead no pipeline:
     - 90-100: Verde (#43A047) — "Excelente"
     - 70-89: Azul (#1E88E5) — "Bom"
     - 50-69: Amarelo (#F9A825) — "Regular"
     - < 50: Vermelho (#E53935) — "Fraco"
   - [ ] Numero do score visivel no badge (ex: "Score 85")
   - [ ] Tooltip ao passar o mouse mostra breakdown (lista de componentes com check/x)

4. **Filtro e Ordenacao**
   - [ ] Dropdown de filtro por range de score (Todos, 90-100, 70-89, 50-69, <50)
   - [ ] Ordenacao por score (maior primeiro como default)
   - [ ] Filtro combinavel com filtros existentes (estado, cidade, busca)

5. **Migration de Dados Existentes**
   - [ ] Script de migration recalcula score de todos os leads existentes no banco
   - [ ] Migration nao altera outros campos do lead
   - [ ] Log da migration: "X leads atualizados, score medio: Y"

6. **Code Quality**
   - [ ] Funcao `calculateLeadScore(lead)` com unit tests
   - [ ] Testes cobrem: lead completo (100), lead minimo (25 — so telefone), lead vazio (0)
   - [ ] Testes para cada componente individual
   - [ ] All tests pass: `npm test`

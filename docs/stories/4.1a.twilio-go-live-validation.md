# Story 4.1a: Twilio Go-Live Validation (UAT Externo)

**Epic:** Module 4 - WhatsApp SDR Automation
**Story ID:** 4.1a
**Status:** Ready for Dev
**Assigned To:** @qa
**Story Points:** 3

---

## Description

Executar validação real de produção assistida da integração WhatsApp/Twilio após disponibilização de credenciais e número oficial, sem alterar regras funcionais já implementadas na Story 4.1.

**User Story:**
> Como PO/QA, quero validar envio e rastreio reais no provider para homologar o módulo WhatsApp com evidência operacional de ponta a ponta.

---

## Acceptance Criteria

### AC1: Credenciais e Número
- [ ] Account SID, Auth Token e número WhatsApp configurados em ambiente de teste controlado
- [ ] Verificação de conexão com provider concluída sem erro

### AC2: Fluxo E2E Real
- [ ] Envio real de mensagem de teste para número válido
- [ ] Atualização de status via webhook (sent/delivered/read ou equivalente)
- [ ] Histórico de mensagens refletido corretamente na API

### AC3: Evidência Operacional
- [ ] Evidências anexadas (logs, payloads webhook, resposta API)
- [ ] Registro de data/hora da validação e ambiente utilizado
- [ ] Risco residual documentado (se houver)

---

## Definition of Done

- [ ] UAT com provider real concluído
- [ ] Critérios AC1-AC3 atendidos
- [ ] Story 4.1 referenciada como fechamento técnico anterior

---

## Dependencies

- Story 4.1 concluída tecnicamente
- Credenciais externas válidas (Twilio/WhatsApp)
- Endpoint de webhook acessível no ambiente de validação

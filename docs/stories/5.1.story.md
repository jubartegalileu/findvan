## Story Summary

Concluir os requisitos P0 do Dashboard com três blocos: 6 KPIs, mini-funil visual clicável e widget de ações urgentes priorizadas.

## User Story

Como founder/operador,
quero visualizar KPIs, estágio de funil e urgências em um único painel,
para agir rápido nos leads com maior impacto comercial.

## Acceptance Criteria

1. **KPIs Expandidos (RF-D1)**
- [x] Dashboard exibe 6 cards: validos, coletas hoje, capturados 24h, contactados, taxa resposta, conversões do mês
- [x] Loading skeleton durante carregamento
- [x] Cadência única de atualização para os 6 cards

2. **Mini-Funil Visual (RF-D2)**
- [x] Exibe 6 estágios com barra proporcional, valor absoluto e percentual
- [x] Rodapé com taxa geral de conversão
- [x] Clique em estágio navega para Leads aplicando filtro correspondente

3. **Ações Urgentes (RF-D3)**
- [x] Widget com prioridades: vermelho > amarelo > verde
- [x] Cada linha mostra quantidade e ação clicável
- [x] Estado vazio mostra "Tudo em dia!"

4. **Compatibilidade e Qualidade**
- [x] Sem quebra de APIs existentes (com alias temporário quando necessário)
- [x] Sem regressão visual nas demais seções do Dashboard
- [x] Testes mínimos de integração dos endpoints do dashboard

## Technical Notes

- **Endpoints alvo:**
  - `GET /api/dashboard/kpis`
  - `GET /api/dashboard/funnel-summary`
  - `GET /api/dashboard/urgent-actions`
- **Integração crítica:** navegação Dashboard -> Leads com filtros consistentes
- **Padrão:** reutilizar componentes visuais e classes de `dashboard.css`

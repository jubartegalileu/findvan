# Story 5.2 — Scraper P0 Completion (Keywords, Feedback Inteligente, Pipeline Transparente)

**Epic:** 5 (Wave 1 P0 Closure)
**Story ID:** 5.2
**Status:** Ready for QA
**Priority:** P0 (Must Have)
**Complexity:** Medium (5 pontos)
**RF:** RF-S1, RF-S2, RF-S3

---

## Story Summary

Concluir os requisitos P0 do Scraper: keywords customizáveis por cidade, feedback inteligente para baixa captura e pipeline visual completo por execução.

## User Story

Como operador,
quero configurar palavras-chave e entender claramente por que uma coleta performou mal,
para ajustar a estratégia rapidamente e aumentar leads novos.

## Acceptance Criteria

1. **Keywords Customizáveis (RF-S1)**
- [x] Campo de keywords multi-valor no formulário de coleta
- [x] Permite múltiplas keywords separadas por vírgula/chips
- [x] Persistência por cidade/UF
- [x] Coleta executa para cada keyword configurada

2. **Feedback Inteligente (RF-S2)**
- [x] Banner amarelo quando execução retorna 0 novos
- [x] Sugestões automáticas de keywords alternativas
- [x] Contador de execuções consecutivas com baixo resultado
- [x] Botão "Editar keywords" no próprio alerta

3. **Pipeline Transparente (RF-S3)**
- [x] Execução exibe 4 estágios: encontrados > válidos > duplicados > novos
- [x] Barra segmentada por etapa
- [x] Tooltip com explicação de duplicados/descartes

4. **Compatibilidade e Qualidade**
- [x] Fluxo atual de coleta continua funcional sem keyword customizada
- [x] Mensagens de erro continuam claras para timeout/rede
- [x] Testes cobrindo casos 0 resultado e múltiplas keywords

## Technical Notes

- **Integrações críticas:**

- **Persistência:** armazenar configuração por cidade/UF sem quebrar payload antigo
- **Padrão:** manter UI atual do Scraper com expansão progressiva

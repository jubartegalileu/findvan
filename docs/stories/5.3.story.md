# Story 5.3 — Leads P0 Completion (Filtros Expandidos + Ações em Lote)

**Epic:** 5 (Wave 1 P0 Closure)
**Story ID:** 5.3
**Status:** Ready for QA
**Priority:** P0 (Must Have)
**Complexity:** Grande (8 pontos)
**RF:** RF-L3, RF-L4

---

## Story Summary

Fechar os gaps P0 de Leads com filtros expandidos completos e ações em lote operacionais, mantendo performance e consistência com score/funil já implantados.

## User Story

Como SDR,
quero filtrar leads com precisão e executar ações em lote,
para operar o funil com velocidade e sem trabalho manual repetitivo.

## Acceptance Criteria

1. **Filtros Expandidos (RF-L3)**
- [x] Filtros disponíveis: estado, cidade, busca, status, score, fonte, data de captura
- [x] Lógica AND entre todos os filtros
- [x] Botão "Limpar filtros"
- [x] Persistência dos filtros na sessão

2. **Ações em Lote (RF-L4)**
- [x] Checkbox por lead + selecionar todos
- [x] Barra de ações visível quando houver seleção
- [x] Contador "X selecionados"
- [x] Ações: alterar status, adicionar campanha, exportar CSV, excluir (soft delete)
- [x] Confirmação para ação destrutiva
- [x] Feedback de sucesso/erro por operação

3. **Compatibilidade e Qualidade**
- [x] Fluxo de edição individual de lead permanece intacto
- [x] Integração com status de funil mantém validações de transição
- [x] Performance aceitável com paginação atual

4. **Testes P0**
- [ ] Testes de filtros combinados
- [x] Testes de ações em lote (happy path + erro)
- [ ] Testes de regressão dos fluxos já entregues (score e funil)

## Technical Notes

- **Endpoints novos/ajustados esperados:**

  - `POST /api/leads/batch/campaign`
  - `POST /api/leads/batch/export` (ou estratégia equivalente)
  - `POST /api/leads/batch/delete` (soft)
- **Integração crítica:** consistência entre filtros do Leads e links vindos do Dashboard
- **Padrão:** manter arquitetura atual do `Leads.jsx` com extração gradual de componentes

## Risks

- **Risco:** ações em lote alterarem registros indevidos por falha de seleção.
- **Mitigação:** confirmação explícita, payload com IDs fechados, retorno detalhado por item.
- **Rollback:** desativar barra batch por feature flag e manter operação individual.

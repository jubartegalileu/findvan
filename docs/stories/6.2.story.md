## API Contract (PO detail)

1. `POST /api/scraper/schedules`
- Payload exemplo:
```json
{
  "state": "SP",
  "city": "Campinas",
  "keywords": ["transporte escolar", "van escolar"],
  "quantity": 50,
  "frequency": "weekly",
  "day_of_week": 1,
  "execution_time": "09:00",
  "is_active": true
}
```

2. `GET /api/scraper/schedules`
- Retorna lista de agendamentos com campos de controle.

3. `PATCH /api/scraper/schedules/{id}`
- Permite atualizar campos e toggle `is_active`.

4. `DELETE /api/scraper/schedules/{id}`
- Remove agendamento.

## UI Behavior (PO detail)

- Tela Scraper deve exibir seção `Agendamentos` com tabela/lista simples.
- Exibir feedback imediato ao atingir limite de 5 ativos.
- Em erro de criação, manter dados preenchidos para correção.

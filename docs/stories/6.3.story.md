# Story 6.3 — Leads UX (RF-L5, RF-L6, RF-L7)

**Epic:** 6 (Wave 2 P1 Expansion)
**Story ID:** 6.3
**Status:** Ready for QA
**Priority:** P1 (Should Have)
**Complexity:** Grande (8 pontos)
**RF:** RF-L5, RF-L6, RF-L7

## Summary

Evoluir UX da tela Leads com cards enriquecidos, sidebar de insights e modal com tabs.

## Acceptance Criteria

1. RF-L5 Cards Enriquecidos
- [x] Card exibe: nome/empresa, cidade/UF/telefone, endereço/fonte, score/status, indicadores de completude e tags.
- [x] Exibir alerta visual de follow-up vencido.
- [x] Destacar lead com resposta recente.
- [x] Ações rápidas no card: `Contactar`, `Ver`, `Menu`.

2. RF-L6 Sidebar Insights
- [x] Sidebar fixa com resumo de contagens (total, válidos, duplicados).
- [x] Conversão por etapa do funil calculada corretamente.
- [x] Distribuição de score por faixas (90-100, 70-89, 50-69, <50).
- [x] Alertas clicáveis aplicam filtros na lista.
- [x] Insights respeitam filtros ativos da tela.

3. RF-L7 Modal com Tabs
- [x] Modal com tabs `Dados`, `Histórico`, `Score`, `Notas`.
- [x] Tab `Dados` abre por padrão com status editável.
- [x] Tab `Histórico` mostra eventos em ordem decrescente.
- [x] Tab `Score` mostra breakdown completo da pontuação.
- [x] Tab `Notas` suporta criar/listar notas.
- [x] Botão `Enviar mensagem` visível em todas as tabs.

4. Compatibilidade e Qualidade
- [x] Fluxos P0 (filtros, seleção em lote, ações batch) permanecem intactos.
- [x] Performance da lista de leads não degrada de forma perceptível.
- [x] Smoke visual sem erros de console/rede.

## API Contract (PO detail)

1. `GET /api/leads/{id}/interactions`
- Retorna timeline para tab `Histórico`.

2. `GET /api/leads/{id}/score`
- Retorna breakdown para tab `Score`.

3. `GET /api/leads/{id}/notes` e `POST /api/leads/{id}/notes`
- Endpoint para CRUD mínimo de notas (listar + criar nesta wave).

## UI Behavior (PO detail)

- Modal deve manter contexto do lead ao trocar tabs.
- Mudança de status no modal deve atualizar card sem refresh completo.
- Sidebar pode ser colapsável em resoluções menores, mantendo acesso aos insights.

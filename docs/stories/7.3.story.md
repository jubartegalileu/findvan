## Summary

Implementar sistema de tags customizáveis em Leads para classificação manual, filtro e operação em lote.

## Acceptance Criteria

1. CRUD mínimo de tags no lead
- [x] Adicionar tags no modal de detalhes (tab `Dados`).
- [x] Remover tags existentes no lead.
- [x] Autocomplete com tags já existentes.

2. Exibição e filtro
- [x] Exibir tags no card do lead.
- [x] Filtro por tag na tela de Leads.
- [x] Filtro combinado com demais filtros (AND logic) permanece funcional.

3. Lote
- [x] Ação em lote `Tagear` funcional para leads selecionados.
- [x] Feedback de sucesso/erro após operação em lote.

4. Compatibilidade e Qualidade
- [x] Fluxos P0/P1 de seleção em lote e status não regressam.
- [x] Performance da lista continua estável em paginação atual.

## API Contract (PO detail)

1. `POST /api/leads/{id}/tags`
2. `DELETE /api/leads/{id}/tags/{tag}`
3. `POST /api/leads/batch/tag`

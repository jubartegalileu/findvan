# Story 8.1 — Environment Hardening (Runtime Local Canônico)

**Epic:** 8 (Wave 4 Platform Stabilization)
**Story ID:** 8.1
**Status:** Done
**Priority:** P0 interno
**Owner:** @dev
**Estimate:** TBD
**Complexity:** Média (5 pontos)

## Summary

Padronizar o runtime local com comandos canônicos para subir/parar/inspecionar backend e dashboard com perfil único de portas, reduzindo ruído operacional e falsos negativos em validações QA.

## Acceptance Criteria

1. Runtime canônico
- [x] Comando único para subir ambiente local canônico (backend + dashboard).
- [x] Perfil fixo documentado: backend `8000`, dashboard `5173`.
- [x] Dashboard sobe com `VITE_API_URL` alinhada ao backend canônico.

2. Operação e observabilidade
- [x] Comando para derrubar ambiente gerenciado sem matar processos externos.
- [x] Comando para status dos serviços gerenciados.
- [x] Comando de health check HTTP para backend e dashboard.
- [x] Logs e PID files centralizados para troubleshooting.

3. Governança de execução
- [x] Documentação operacional versionada no repositório.
- [x] Scripts publicados no `package.json` para uso padrão do time.

4. Compatibilidade
- [x] Sem quebra de contratos de API/UI entregues em Waves 1-3.
- [x] Mantidos scripts legados (`dev:*`) para uso complementar.

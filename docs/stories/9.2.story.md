# Story 9.2 — Contract Versioning v1.1

**Epic:** 9 (Wave 5 Integration Rollout)
**Story ID:** 9.2
**Status:** Done
**Priority:** P1
**Complexity:** Média (5 pontos)

## Summary

Implementar evolução de contrato de integração para `v1.1.0` com estratégia de compatibilidade retroativa, mantendo `v1.0.0` como default.

## Acceptance Criteria

1. Versionamento
- [x] Endpoint de contratos aceita versão por query param.
- [x] `v1.0.0` permanece default para clientes legados.
- [x] `v1.1.0` disponível por opt-in.

2. Compatibilidade
- [x] Evolução apenas aditiva em `v1.1.0`.
- [x] Matriz de compatibilidade publicada na resposta.
- [x] Versão inválida retorna erro explícito (`400`).

3. Governança
- [x] Documentação de versionamento publicada.
- [x] Testes cobrindo default, opt-in e erro de versão.

## Tasks

- [x] Evoluir `contracts.py` com versões `v1.0.0` e `v1.1.0`
- [x] Atualizar endpoint `GET /api/integrations/contracts`
- [x] Adicionar testes de API e contrato
- [x] Documentar estratégia de versionamento

## Definition of Done

- [x] Versionamento `v1 -> v1.1` implementado sem quebra
- [x] Compatibilidade retroativa coberta por testes
- [x] `npm run lint`, `npm run typecheck`, `npm test` verdes

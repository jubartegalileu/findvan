## Functional Validation (code + contract)

1. Contratos de integração
- UI consulta `GET /api/integrations/contracts` ao carregar e ao revalidar status.
- Exibe provider solicitado, provider efetivo, fallback e versões de contrato.

2. Envio incremental
- UI envia para `POST /api/integrations/messaging/send` com payload:
  - `lead_id`, `to`, `content`, `dry_run`, `provider`
- `dry_run` padrão ativo na interface.
- Feedback de sucesso e erro operacional renderizado no painel.

3. Fallback operacional
- Interface informa fallback quando provider solicitado difere do efetivo.
- Fluxo não bloqueia uso quando provider real não está configurado.

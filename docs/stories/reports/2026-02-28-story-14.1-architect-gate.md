# Architect Gate — Story 14.1 (Async Retention Jobs)

**Date:** 2026-02-28  
**Agent:** @architect  
**Status:** PASS

## Scope Reviewed

- Remoção de expurgo de dados do request path de endpoints operacionais.
- Introdução de job assíncrono periódico para retenção de receipts/activity.
- Compatibilidade de contrato e robustez operacional no ciclo de startup/shutdown.

## Architectural Validation

1. Async retention design
- Serviço dedicado `retention_jobs_service` implementa ciclo periódico com worker daemon, controle de start/stop e estado operacional.
- Job executa expurgo de receipts/activity com janelas configuráveis por ambiente e registra métricas de execução (run_count, fail_count, last_deleted, timestamps).
- Execução imediata após startup estabelece baseline operacional sem acoplamento ao request path.

2. Request path hardening
- Endpoints `GET /api/integrations/messaging/receipts` e `GET /api/activity` não executam mais expurgo inline.
- Contrato de resposta permaneceu compatível (`events`, `applied_limit`, `retention_pruned`), com `retention_pruned=0` para preservar semântica.
- Endpoints adicionais de status do job (`/api/integrations/messaging/retention-job` e `/api/activity/retention-job`) são aditivos.

3. Reliability and lifecycle
- Startup do app inicia job de retenção com telemetria de estado; shutdown encerra job explicitamente.
- Health check agora inclui sinal mínimo de retenção (`enabled`, `running`, `last_success_at`).
- Falhas no ciclo de retenção não propagam erro para fluxos principais de API.

4. Evidence and tests
- Testes de API atualizados para novo comportamento sem expurgo inline e para status do job.
- Testes dedicados de serviço (`test_retention_jobs_service.py`) cobrem ciclo e modo desabilitado.
- Gates técnicos (`lint`, `typecheck`, `test`) verdes.

## Residual Risks (non-blocking)

- Estado do job é em memória de processo; para multi-instância, recomenda-se coordenação distribuída ou execução em worker dedicado.

## Verdict

- Story 14.1 aprovada no gate @architect.
- Sem bloqueio arquitetural para avanço ao gate @qa.

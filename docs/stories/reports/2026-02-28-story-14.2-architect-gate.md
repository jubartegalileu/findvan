# Architect Gate — Story 14.2 (External Operational Alerting)

**Date:** 2026-02-28  
**Agent:** @architect  
**Status:** PASS

## Scope Reviewed

- Contrato de alerta operacional externo para eventos críticos de SLO.
- Robustez de fallback local, retry e supressão de ruído.
- Compatibilidade de contrato em endpoints de dashboard.

## Architectural Validation

1. External alert contract
- Serviço `alerts_service` define payload versionado (`contract_version=1.0.0`) com severidade, métrica, janela e timestamp.
- Endpoint `GET /api/dashboard/weekly-performance` mantém semântica original e adiciona metadado aditivo `alert_dispatch`.
- Endpoints aditivos de operação (`POST /api/dashboard/alerts/dispatch`, `GET /api/dashboard/alerts/status`) preservam backward compatibility.

2. Fallback reliability
- Cenário sem `ALERT_WEBHOOK_URL` usa fallback local sem impactar fluxo principal.
- Envio externo executa em thread assíncrona, evitando bloqueio de request path.
- Failures de webhook não quebram API; estado local captura erro e evidência de fallback.

3. Alert noise control
- Supressão por cooldown implementada por chave (`event_type|severity|source`).
- Retry controlado por configuração de ambiente e timeout dedicado.
- Estado operacional oferece visibilidade de `suppressed_count`, `fallback_count`, `sent_count` e últimos eventos.

4. Evidence and tests
- Testes de serviço cobrem contrato, fallback e supressão.
- Testes de API cobrem dispatch/status e integração no weekly performance.
- Gates técnicos (`lint`, `typecheck`, `test`) verdes.

## Residual Risks (non-blocking)

- Estado de supressão/fallback é em memória de processo; para multi-instância recomenda-se store compartilhado.

## Verdict

- Story 14.2 aprovada no gate @architect.
- Sem bloqueio arquitetural para avanço ao gate @qa.

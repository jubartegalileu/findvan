## Architectural Validation

1. Predictive model design
- `operational_telemetry_service` adiciona `PREDICTIVE_RISK_MODEL_VERSION` e cálculo determinístico de score 0-100 por componente.
- Modelo utiliza sinais de incidentes + guardrails + métricas operacionais (retention/alerting).
- Resultado inclui probabilidade e severidade prevista (`low/medium/high/critical`).

2. Contract compatibility
- Endpoint aditivo `GET /api/dashboard/predictive-risk` entregue com `limit/offset` e metadado de versão do modelo.
- Nenhum campo existente foi removido de telemetria/guardrails.
- Falha no cálculo fica confinada ao endpoint preditivo, sem impacto em rotas principais.

3. UI alignment
- Dashboard consome e exibe risco preditivo por componente com recomendação operacional.
- Priorização visual consistente com a severidade prevista do backend.

4. Validation evidence
- Testes de API atualizados em `test_dashboard_api.py` cobrindo endpoint de risco preditivo.
- Quality gates verdes no ciclo: `npm run lint`, `npm run typecheck`, `npm test`.

## Residual Risks (non-blocking)

- Modelo inicial é heurístico; recomenda-se calibrar pesos conforme dados reais das próximas waves.
- Necessário monitorar estabilidade de score para evitar oscilação excessiva em baixa amostragem.

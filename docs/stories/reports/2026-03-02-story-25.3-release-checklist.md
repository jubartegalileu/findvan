## Go/No-Go Criteria

## GO
- Todos os quality gates verdes
- Sem erro 500 recorrente nos endpoints SDR/Pipeline
- Fluxos críticos SDR/Funil/Leads operacionais

## HOLD
- Falha em quality gate crítico (lint/typecheck/test)
- Erro de contrato API (payload incompatível no frontend)
- Regressão funcional em ações principais de SDR/Funil

## Rollback Checklist

1. Identificar commit alvo estável anterior no `main`.
2. Reverter commit(s) de release (ou rollback de deploy conforme ambiente).
3. Validar rapidamente:
- [ ] `/health`
- [ ] `/api/leads`
- [ ] Dashboard abre sem crash
4. Registrar incidente e causa provável em relatório QA.

## Known Residual Risks

- Warnings de `act(...)` em testes de SDR/Funil não quebram a suíte, mas reduzem sinal de qualidade.
- Tratar como item prioritário de estabilização de teste antes de expansão de features.
